# תוכנית לפיתוח דף העלאת פרויקט פורטפוליו

## 🎯 מטרה
יצירת דף העלאת פרויקט מקיף ומקצועי שיהיה ליבת הפלטפורמה, המאפשר למשתמשים לשתף את העבודות והפרויקטים שלהם עם הקהילה.

---

## 📋 שלב 1: מבנה הדף והניווט

### 1.1 דף נפרד או דיאלוג?
**החלטה: דף נפרד (`/portfolios/upload`)**
- ✅ יותר מקום לתוכן
- ✅ חוויית משתמש טובה יותר
- ✅ אפשרות לשמירת טיוטה
- ✅ תצוגה מקדימה מלאה

### 1.2 ניווט
- כפתור "הוסף פרויקט חדש" יוביל לדף חדש
- לחץ אחורה/ביטול יחזיר לדף תיקי עבודות
- שמירה כטיוטה - אפשרות לחזור ולהמשיך מאוחר יותר

## 📝 שלב 2: שדות הטופס

### 2.1 מידע בסיסי (חובה)
- [x] **שם הפרויקט** - TextField (כבר קיים)
- [ ] **תיאור מפורט** - TextArea (גדול, עם תמיכה ב-markdown?)
- [ ] **קטגוריה** - Select/Dropdown עם רשימה מוגדרת:
  - עיצוב פנים
  - אדריכלות
  - אומנות
  - צילום
  - טכנולוגיה/קוד
  - נגרות
  - גינון
  - בישול
  - אחר
- [ ] **תגיות** - Multi-select או chips (למשל: React, עיצוב, קיבוץ, קהילה)

### 2.2 מדיה (חובה - לפחות תמונה אחת)
- [ ] **תמונה ראשית** - Upload component עם:
  - Drag & Drop
  - תצוגה מקדימה
  - Crop/Resize
  - וולידציה (גודל, פורמט)
- [ ] **גלריית תמונות נוספות** - Multiple upload:
  - עד 10 תמונות
  - סדר התמונות (drag to reorder)
  - תצוגה מקדימה
- [ ] **וידאו** (אופציונלי):
  - URL (YouTube, Vimeo)
  - או העלאת קובץ

### 2.3 פרטים נוספים (אופציונלי)
- [ ] **תאריך ביצוע** - Date picker
- [ ] **טכנולוגיות/כלים** - Multi-select chips:
  - React, Vue, Angular
  - Photoshop, Figma
  - עץ, מתכת, וכו'
- [ ] **קישורים חיצוניים**:
  - GitHub repository
  - אתר הפרויקט
  - Behance/Dribbble
  - קישור מותאם אישית
- [ ] **שיתופי פעולה** - רשימת משתתפים נוספים (חברי קהילה)
- [ ] **מיקום** - איפה בוצע הפרויקט (אופציונלי)

## 🎨 שלב 3: עיצוב ו-UX

### 3.1 מבנה הדף
```
┌─────────────────────────────────────┐
│  [← חזרה]  העלאת פרויקט חדש        │
├─────────────────────────────────────┤
│                                     │
│  [תצוגה מקדימה]  │  [טופס]         │
│                  │                  │
│                  │  שלב 1: מידע     │
│                  │  שלב 2: מדיה    │
│                  │  שלב 3: פרטים   │
│                  │                  │
│                  │  [שמור כטיוטה]  │
│                  │  [פרסם]         │
└─────────────────────────────────────┘
```

### 3.2 Stepper/Wizard (מומלץ)
- **שלב 1: מידע בסיסי** - שם, תיאור, קטגוריה
- **שלב 2: מדיה** - תמונות, וידאו
- **שלב 3: פרטים נוספים** - קישורים, תאריכים, וכו'
- **שלב 4: תצוגה מקדימה** - סיכום לפני פרסום

### 3.3 תצוגה מקדימה
- תצוגה בזמן אמת של איך הפרויקט יראה
- Card preview כמו בדף הראשי
- אפשרות לערוך מכל שלב

### 3.4 וולידציה
- שדות חובה מסומנים בבירור
- הודעות שגיאה ברורות
- אינדיקטור התקדמות
- בדיקת גודל קבצים לפני העלאה

## 🔧 שלב 4: פונקציונליות טכנית

### 4.1 ניהול State
- React state management (useState/useReducer)
- שמירת טיוטה ב-localStorage
- שחזור טיוטה בעת חזרה לדף

### 4.2 העלאת קבצים
- **אופציה 1**: שירות ענן (Cloudinary, AWS S3)
- **אופציה 2**: Backend API עם storage מקומי
- Progress indicator
- Error handling

### 4.3 API Integration
```
POST /api/portfolios
- Create new portfolio
- Upload images
- Save draft

GET /api/portfolios/drafts
- Get user drafts

PUT /api/portfolios/:id
- Update portfolio
```

## 🎨 Design Considerations

### צבעים
- כפתור פרסום: `#2b868a` (כמו כפתור הוספה)
- שדות חובה: סימון ברור
- שגיאות: אדום עדין
- הצלחה: ירוק

### Icons
- PlusIcon - הוספת תמונה

- DragHandleIcon - שינוי סדר

- CalendarIcon - תאריך

## 📝 הערות נוספות

1. **נגישות**: תמיכה ב-keyboard navigation, ARIA labels
2. **ביצועים**: Lazy loading לתמונות, אופטימיזציה של תמונות
3. **אבטחה**: וולידציה בצד שרת, בדיקת סוגי קבצים
4. **UX**: הודעות הצלחה, loading states, error recovery
